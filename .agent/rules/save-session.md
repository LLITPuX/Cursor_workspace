# Команда /ss

Коли користувач використовує команду `/ss`:

## Автоматичне збереження сесії

1. **Збір історії сесії:**
   - Збери ВСІ повідомлення з поточної сесії Cursor
   - Включи всі "Запит користувача" та "Відповідь"
   - Збережи структуру "Аналіз та дії" для кожного відповіді
   - **КРИТИЧНО:** Зберігай відповіді СЛОВО В СЛОВО. Ніколи не скорочуй і не роби саммарі відповідей.
   - НЕ обмежуй розмір - зберігай повністю

2. **Визначення теми:**
   - Проаналізуй ВСЕ вміст сесії (не тільки перший запит)
   - Виділи ключові концепції, технології, задачі з усіх повідомлень
   - Сформуй коротку тему (3-5 слів, максимум 50 символів)
   - Використовуй snake_case для імені файлу
   - Приклади: "db_command_implementation", "python_issue_diagnosis", "falkordb_planning"

3. **Генерація часу:**
   - Отримай поточний час: використай точну дату та час
   - Формат для відображення: "20 січня 2026, 14:30:00"
   - Формат для імені файлу: `2026-01-20_14-30-00` (HH-MM-SS)

4. **Створення "Підсумок сесії":**
   - Автоматично визнач обговорені теми з усієї сесії
   - Виділи виконані завдання (по наявності ✅ або завершених дій)
   - Сформуй коротке резюме результатів
   - Завжди створюй цей розділ автоматично

5. **Формат файлу:**
   ```markdown
   # Сесія: <автоматично визначена тема>

   **Дата:** <дата>, <час>
   **Тема:** <повна тема>

   ---

   ## Запит користувача #1

   ```
   <текст запиту>
   ```

   ### Аналіз та дії

   <мій аналіз та дії>

   ### Відповідь #1

   <моя відповідь>

   ---

   ## Підсумок сесії

   ### Обговорені теми:
   1. <тема 1>
   2. <тема 2>

   ### Виконані завдання:
   1. ✅ <завдання 1>
   2. ✅ <завдання 2>

   ### Результат:

   <коротке резюме результатів>

   ---

   **Кінець сесії**
   ```

6. **Збереження файлу:**
   - Ім'я: `saved_sessions/session_YYYY-MM-DD_HH-MM-SS_<тема>.md`
   - Форматування за шаблоном вище
   - Збереження через інструмент `write`

## Важливо

- **Точний час обов'язковий** - для відстеження хронології кількох сесій в день
- **Автоматичне визначення теми** - по всьому вмісту сесії, не з першого запиту
- **Автоматичний "Підсумок сесії"** - завжди створювати
- **Не обмежувати розмір** - зберігати повністю
- **Формат має бути сумісним з `/db`** - файл має коректно парситися командою `/db`

## Приклади використання

```bash
# Просто напиши команду
/ss

# Команда автоматично:
# 1. Збере всю історію сесії
# 2. Визначить тему по вмісту
# 3. Додасть точний час
# 4. Створить "Підсумок сесії"
# 5. Збереже файл у saved_sessions/
```

## Формат імені файлу

- `session_2026-01-27_14-30-00_db_command_implementation.md`
- Дата: `YYYY-MM-DD`
- Час: `HH-MM-SS` (для сумісності з файловою системою)
- Тема: `snake_case`, максимум 50 символів

## Визначення теми

Аналізуй ВСЕ вміст сесії:
- Ключові слова та концепції
- Технології та інструменти
- Виконані задачі
- Основна мета сесії

НЕ використовуй тільки перший запит - аналізуй всю сесію для точного визначення теми.
